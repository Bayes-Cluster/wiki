Web Portal

Code-Hub is a intergradted web application for USBC cluster, which allows user to upload, modify and delete files; submit, check, hold, kill SLURM jobs as well, which decreate the dependence of prior knowledge in LINUX and HPC.

 

JupyterHub

Jupyterhub provides a multi-user hub for spawning, managing, and proxying multiple instances of single-user Jupyter notebook servers. At USBC, you authenticate to the JupyterHub instance we manage using your USBC credentials.

Server Option

After the user login to the system, we have provide multiple options with different configuration for user to choose:

  • Control Node: user can use the intergradte extension, e.g. web-based VS Code, SLURM extension and etc.

  • CPU Node: user can use the Compute Node with a basic python environment and R environment

  • GPU Node: user can use the GPU Node with public conda environment(pytorch 1.9.0, Tensorflow-GPU 2.5.0)

 

This service is based on the batchspawner, where it provide start servers in batch scheduled systems.

Stop the Spawner

To stop the server, go to the upper-left corner and choose File, choose Hub Control Panel and click Stop My Server
